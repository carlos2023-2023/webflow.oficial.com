WOMPI — PASO A PASO
1

Crear cuenta en Wompi

Ve a comercios.wompi.co → Regístrate con RUT o cédula → Activa tu cuenta (puede tomar 1-2 días hábiles para producción).

2

Obtener llaves de API

En el dashboard de Wompi → Configuración → Llaves de API. Copia tu llave pública (pub_test_xxx para pruebas).

3

Agregar el widget al sitio (opción más simple)

Pega este código en tu index.html donde quieras el botón de pago:

<!-- Botón de pago Wompi — pegar en index.html --> <form> <script src="https://checkout.wompi.co/widget.js" data-render="button" data-public-key="pub_test_TU_LLAVE_PUBLICA" data-currency="COP" data-amount-in-cents="50000000" <!-- $500.000 COP --> data-reference="WEBFLOW-001" data-redirect-url="https://tudominio.com/gracias" ></script> </form>
4

Opción avanzada: API REST (monto dinámico)

Para cobrar el monto exacto del proyecto, usa la API desde tu backend:

// Node.js / Express — crear transacción Wompi const axios = require('axios'); async function crearPagoWompi(monto, referencia, email) { const res = await axios.post( 'https://sandbox.wompi.co/v1/transactions', { amount_in_cents: monto * 100, // $500K → 50000000 currency: 'COP', customer_email: email, reference: referencia, payment_method: { type: 'PSE', user_type: 0, user_legal_id_type: 'CC', user_legal_id: '123456', financial_institution_code: '1022' } }, { headers: { Authorization: `Bearer TU_LLAVE_PRIVADA` } } ); return res.data; // contiene redirect_url → envía al usuario }
5

Webhook — recibir confirmación de pago

Wompi notifica tu servidor cuando el pago es aprobado. Configura la URL en tu dashboard:

// Endpoint que Wompi llama automáticamente al pagar app.post('/webhook/wompi', (req, res) => { const { event, data } = req.body; if (event === 'transaction.updated' && data.transaction.status === 'APPROVED') { const ref = data.transaction.reference; const monto = data.transaction.amount_in_cents / 100; // Aquí: guardar en BD, enviar email, marcar proyecto como pagado console.log(`Pago recibido: $${monto} COP — Ref: ${ref}`); } res.sendStatus(200); });

🔗 Documentación oficial: docs.wompi.co
🧪 Sandbox: sandbox.wompi.co (pruebas sin dinero real)
📞 Soporte: soporte@wompi.co

EPAYCO — PASO A PASO
1

Crear cuenta

Regístrate en epayco.co → Activa tu cuenta de comercio → Obtén P_CUST_ID_CLIENTE y P_KEY desde el dashboard.

2

Integrar el script

<script src="https://checkout.epayco.co/checkout.js" class="epayco-button" data-epayco-key="TU_PUBLIC_KEY" data-epayco-amount="500000" data-epayco-name="Proyecto Web" data-epayco-description="Desarrollo sitio web" data-epayco-currency="cop" data-epayco-country="co" data-epayco-test="true" data-epayco-email-billing="cliente@correo.com" ></script>
🔗 Docs: docs.epayco.co
PAYU — PASO A PASO
1

Crear cuenta de comercio

Solicita tu cuenta en colombia.payu.com. Necesitas RUT, cuenta bancaria y documentos del negocio. Aprobación en 3-5 días hábiles.

2

Integrar PayU Checkout

// Formulario de pago PayU (Colombia) <form method="post" action="https://checkout.payulatam.com/ppp-web-gateway-payu/"> <input name="merchantId" value="TU_MERCHANT_ID"/> <input name="accountId" value="TU_ACCOUNT_ID"/> <input name="description" value="Proyecto Webflow CO."/> <input name="referenceCode" value="WF-001"/> <input name="amount" value="500000"/> <input name="currency" value="COP"/> <input name="signature" value="MD5_FIRMA"/> <!-- generada en backend --> <input name="test" value="1"/> <button type="submit">Pagar con PSE</button> </form>
🔗 Docs: developers.payulatam.com
MERCADO PAGO — PASO A PASO
1

Crear cuenta y aplicación

Ve a mercadopago.com.co → Tus integraciones → Crea una app → Obtén ACCESS_TOKEN y PUBLIC_KEY.

2

Checkout Pro (más fácil)

// Backend Node.js — crear preferencia de pago const { MercadoPagoConfig, Preference } = require('mercadopago'); const client = new MercadoPagoConfig({ accessToken: 'TU_ACCESS_TOKEN' }); const pref = new Preference(client); const result = await pref.create({ body: { items: [{ title: 'Proyecto Webflow CO.', quantity: 1, unit_price: 500000 }], back_urls: { success: 'https://tudominio.com/gracias', failure: 'https://tudominio.com/error' }, auto_return: 'approved' }}); // Redirige al usuario a: result.init_point
🔗 Docs: mercadopago.com.co/developers
